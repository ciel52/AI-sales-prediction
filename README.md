# 販売予測AIエージェント デモ

イベント・観光分野を想定した **翌日の売上金額予測** のデモ（試作）です。
実データが無い段階で「AI（機械学習）が実際に学習・予測できるか」を体感・検証することを目的としています。

詳細な仕様は [`要件定義.md`](./要件定義.md) を参照してください。

## 構成

```
.
├── 要件定義.md          # デモの要件定義書
├── README.md            # 本ファイル
├── requirements.txt     # 依存パッケージ
├── demo.ipynb           # デモ本体（実行しながら学習・予測・評価を確認）
└── src/
    └── synthetic_data.py  # 合成データ生成モジュール
```

## セットアップ

```bash
# 仮想環境を作成して依存をインストール
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## 実行方法

### 方法1: ノートブックを開いて実行（おすすめ）

```bash
source .venv/bin/activate
jupyter notebook demo.ipynb
```

上から順にセルを実行すると、合成データ生成 → 特徴量作成 → 学習 → 予測 → 精度評価までの流れを確認できます。

### 方法2: コマンドで一括実行

```bash
source .venv/bin/activate
jupyter nbconvert --to notebook --execute --inplace demo.ipynb
```

## デモの結果（参考値）

検証期間（直近90日）の精度（合成データ・seed=42）:

| モデル | MAPE | ピーク日MAPE |
|---|---|---|
| 勾配ブースティング（本命） | 約10% | 約8% |
| ベースライン（前年同日／移動平均） | 約31〜33% | 約25〜29% |

機械学習モデルが、データに埋め込んだ傾向（曜日・季節・祝日・天候・SNS先行効果）を学習し、単純ベースラインを大きく上回ることを確認できます。

## 本番フェーズへの発展

1. `src/synthetic_data.py` の合成データを **実データ読み込み** に置き換える
   （売上実績CSV、気象庁の過去天候データ、SNS取得結果）
