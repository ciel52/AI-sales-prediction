# 来店・売上予測AI（小売・サービス業 ポイントシステム）

小売・サービス業の **ポイントシステム** の購買情報を中心に、天候・人流・SNS などを組み合わせて
**来店予測（主）→ 売上予測（従）** を行う分析AIプロジェクトです。

実データが揃う前に、**擬似データ** で「良い精度の分析ができるか」を検証し、
分析の型（プロンプト／分析パイプライン）を確立することを当面の MVP とします。

詳細な仕様・論点は [`要件定義.md`](./要件定義.md) を参照してください。

### フェーズ1 成果物
- `src/retail_synthetic_data.py`：1店舗・2年分の日次データを生成。来店客数に
  曜日・季節・祝日・給料日・天候・人流・SNS先行・販促（ポイント◯倍/特売/クーポン）の関係を埋め込み済み。
- `phase1_data.ipynb`：生成データの可視化と、埋め込んだ傾向の定量確認。
- `data/retail_synthetic.csv`：フェーズ2でそのまま読み込めるデータ。

> `demo.ipynb` / `src/synthetic_data.py` は、機械学習による予測の型が機能することを確認した
> **技術検証用のPoC**（題材：イベント・観光の売上予測）です。

## 技術検証PoCの実行（参考）

```bash
source .venv/bin/activate
jupyter notebook demo.ipynb
```

PoC の検証結果（合成データ・seed=42、直近90日検証）：

| モデル | MAPE | ピーク日MAPE |
|---|---|---|
| 勾配ブースティング | 約10% | 約8% |
| ベースライン（前年同日／移動平均） | 約31〜33% | 約25〜29% |

機械学習が、データに埋め込んだ傾向を学習し単純ベースラインを大きく上回ることを確認済み。
この型を来店予測に展開していきます。

### LLM施策提案について（フェーズ5）

`phase5_llm_advisor.ipynb` は、予測結果を店長向けの具体的な施策提案に翻訳します。
APIキーがあれば実際のLLMが、無ければ同じ文脈からルールベースの下書きが提案を生成します。

```bash
export OPENAI_API_KEY="sk-..."   # 設定するとLLMが提案を生成（任意）
pip install openai               # 未インストールなら
```

## フェーズ2・3の結果（参考値）

検証期間（直近90日）の精度（擬似データ・seed=42）：

| 予測 | 手法 | MAPE |
|---|---|---|
| 来店客数 | 勾配ブースティング | 約9% |
| 来店客数 | ベースライン（曜日平均／前年同日） | 約21〜29% |
| 売上 | 来店予測 × 客単価予測 | 約10% |
| 売上 | ベースライン（曜日平均） | 約23% |

来店予測がベースラインを大きく上回り、「来店×客単価」の分解方式が売上の直接予測よりも良い精度を確認。

### フェーズ7（任意）：継続学習・施策最適化の方針

定期実行の中で「使うほど賢くなる」仕組み。用途で適した技術が分かれる。

| 用途 | 適した技術 | 補足 |
|---|---|---|
| 予測精度の向上 | 定期再学習＋ドリフト検知（＝強化学習ではない） | 予測は教師あり学習のため |
| 施策提案の改善 | 強化学習／**文脈付きバンディット** | クーポン最適化などに適合 |

- 前提：**施策の実行結果（報酬）を記録する効果測定ログ**が必要（フェーズ6と連携）
- 推奨ステップ：効果ログ基盤 → 再学習ループ → A/B・効果測定 → 文脈付きバンディット → 本格RL
- 詳細は [`要件定義.md`](./要件定義.md) の「9.5 フェーズ7」を参照
